Class Structure

A class in Python is defined with the class keyword. Let's see how to create our Pokemon class:

Python Code

class Pokemon:
    # Constructor - runs when we create an object
    def __init__(self, name, ptype, hp_max):
        self.name = name
        self.ptype = ptype
        self.hp_max = hp_max
        self.hp_actual = hp_max

# Create an object (instance) of the class
pikachu = Pokemon("Pikachu", "Electric", 35)
print(pikachu.name)  # Output: Pikachu

Breaking Down the Code

class Pokemon:
Defines a new class called Pokemon. It's like saying "I'm going to create a new type of thing".
def __init__(self, ...):
The constructor or initializer. It runs automatically when you create an object. This is where you define the initial attributes.
self
Represents the current instance of the object. It's like saying "this specific Pok√©mon".
self.name = name
Creates an attribute for the object. Each Pok√©mon will have its own name.

Step 1: We define the Pokemon class with its basic attributes

Step 2: The __init__ receives the parameters and assigns them as attributes

Step 3: We create an object called pikachu with specific values

Step 4: Now we can access the attributes with pikachu.name

Attributes and Methods

Learn how to add methods (actions) to your classes

Adding Behavior

Methods are functions that belong to a class. They define what an object can do:

Python Code

class Pokemon:
    def __init__(self, name, ptype, hp_max):
        self.name = name
        self.ptype = ptype
        self.hp_max = hp_max
        self.hp_actual = hp_max
    
    # Method: an action the Pokemon can perform
    def take_damage(self, damage):
        self.hp_actual = max(0, self.hp_actual - damage)
        return f"{self.name} took {damage} damage!"
    
    def is_alive(self):
        return self.hp_actual > 0

pikachu = Pokemon("Pikachu", "Electric", 35)
print(pikachu.take_damage(10))  # Pikachu took 10 damage!
print(pikachu.is_alive())  # True

Types of Methods

Instance Methods
Work with data from the specific object. Always receive self as first parameter. Example: take_damage()
State-Modifying Methods
take_damage() changes the value of hp_actual. The object is modified.
Query Methods
is_alive() only reads data without modifying anything. Returns information.

Why use methods?

Encapsulation: The take_damage() method ensures that HP is never negative using max(0, ...)

Reusability: You can call pikachu.take_damage(10) as many times as you need

Clarity: Code is more readable: pikachu.is_alive() vs pikachu.hp_actual > 0

Inheritance and Polymorphism

Create classes that inherit from other classes

What is Inheritance?

Inheritance allows you to create a new class based on an existing one. The new class (child) gets all the attributes and methods of the original class (parent).

Python Code

class Movement:
    def __init__(self, name, power, ptype, accuracy):
        self.name = name
        self.power = power
        self.ptype = ptype
        self.accuracy = accuracy

class CombatPokemon(Pokemon):
    def __init__(self, name, ptype, hp_max, movements=[]):
        super().__init__(name, ptype, hp_max)
        self.movements = movements
    
    def attack(self, movement):
        return f"{self.name} uses {movement.name}!"

thunderbolt = Movement("Thunderbolt", 90, "Electric", 100)
pikachu = CombatPokemon("Pikachu", "Electric", 35, [thunderbolt])
print(pikachu.attack(thunderbolt))

Key Concepts

class CombatPokemon(Pokemon):
CombatPokemon inherits from Pokemon. It automatically gets all its attributes and methods.
super().__init__(...)
Calls the parent class (Pokemon) constructor to initialize the basic attributes.
Functionality Extension
CombatPokemon adds new attributes (movements) and methods (attack) without modifying Pokemon.

Advantages of Inheritance

Code Reuse: You don't need to rewrite name, type, hp_max

Specialization: CombatPokemon is a Pokemon with extra capabilities

Maintainability: If you change Pokemon, CombatPokemon updates automatically

Encapsulation

Protect your class data with private attributes

Why Encapsulate?

Encapsulation hides the internal details of a class and only exposes what's necessary. This prevents errors and makes your code more secure.

Python Code

class Pokemon:
    def __init__(self, name, ptype, hp_max):
        self.name = name
        self.ptype = ptype
        self._hp_max = hp_max  # _ indicates protected
        self.__hp_actual = hp_max  # __ makes it private
    
    # Property: allows reading hp_actual in a controlled way
    @property
    def hp(self):
        return self.__hp_actual
    
    # Setter: permite modificar hp_actual con validaci√≥n
    @hp.setter
    def hp(self, valor):
        if valor < 0:
            self.__hp_actual = 0
        elif valor > self._hp_max:
            self.__hp_actual = self._hp_max
        else:
            self.__hp_actual = valor

Access Levels

Public: name
Can be accessed from anywhere. Usage: pokemon.name
Protected: _hp_max
Convention to indicate "use internally only". Python doesn't technically restrict it.
Private: __hp_actual
Python modifies the name to make external access difficult. Truly protected.
@property and @setter
Allow accessing private attributes with validation. Looks like an attribute but is a method.

Practical Benefits

Validation: The setter ensures that HP is never negative or greater than the maximum

Control: You can change the internal implementation without affecting external code

Example: pikachu.hp = -10 automatically becomes pikachu.hp = 0

Composition

Combine multiple objects to create complex structures

Objects Within Objects

Composition means that an object contains other objects. It's a "has-a" relationship instead of "is-a" like inheritance.

Python Code

class Movement:
    def __init__(self, name, power, ptype, accuracy):
        self.name = name
        self.power = power
        self.ptype = ptype
        self.accuracy = accuracy

class Pokemon:
    def __init__(self, name, ptype, hp_max, movements=None):
        self.name = name
        self.ptype = ptype
        self.hp_max = hp_max
        self.hp_actual = hp_max
        self.movements = movements if movements is not None else []  # Pokemon HAS movements

class Trainer:
    def __init__(self, name):
        self.name = name
        self.team = []  # Trainer HAS Pokemon
    
    def catch_pokemon(self, pokemon):
        if len(self.team) < 6:
            self.team.append(pokemon)
            return f"{self.name} caught {pokemon.name}!"
    
    def view_team(self):
        for pokemon in self.team:
            print(f"- {pokemon.name} ({pokemon.ptype})")

# Create movements
thunderbolt = Movement("Thunderbolt", 90, "Electric", 100)

# Create Pokemon with movements
ash = Trainer("Ash")
pikachu = Pokemon("Pikachu", "Electric", 35, [thunderbolt])
ash.catch_pokemon(pikachu)
ash.view_team()

Composition vs Inheritance

Inheritance (is-a)
CombatPokemon IS A Pokemon. Use when there's a specialization relationship.
Composition (has-a)
Pokemon HAS movements. Trainer HAS Pok√©mon. Use when combining independent objects.
Flexibility
With composition you can easily change a Pok√©mon's movements or a Trainer's team.
